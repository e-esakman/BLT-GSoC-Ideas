**BLT-Smart-Reporter AI Explainer**
This project modernizes and stabilizes the existing OWASP BLT browser extension, restores currently broken functionality (including trademark search and reporting workflows), and upgrades it into a reliable, cross-browser, AI-assisted vulnerability reporting tool.

**Core Objectives**
This project focuses first on:
- restoring existing features
- fixing reliability issues
- adding smart reporting
- AI-assisted explanations


**Proposed Timeline**
Phase 1: Audit & Repair (Weeks 1–3 | ~90 hours)
- identify broken flows
- fix trademark search scanner
- add logging
- improve job search functionality
- write baseline tests

Phase 2: Modernization (Weeks 4–8 | ~120 hours)
- service worker conversion
- TypeScript refactor
- permissions cleanup
- CSP hardening
- project structure cleanup
- metadata auto-capture
- screenshot capture
- CI/CD + e2e tests
- add bug reporting

Phase 3: AI Assistance & Intelligence (Weeks 10–11 | ~50 hours)
- “Explain vulnerability”
- LLM integration
- auto-filled descriptions

Phase 4: Polish & Release (Week 12 | ~20 hours)
- testing
- documentation
- demo video

**Benefits**
- Restores currently broken features
- Makes extension usable again
- Cross-browser support
- Bug reporting directly from extension
